# Backlog

> **Important:** This file is kept intentionally minimal.
> The single source of truth for the backlog is the **GitHub Projects V2 Board** created by `gh-repo-bootstrap`.

To view the active backlog, sprint plans, and current status of all tickets, please visit the Projects tab in this repository.

## Adding Items to the Backlog

1. Do **not** edit this file manually.
2. Open a new **GitHub Issue** using the appropriate template (Feature, Bug, Compliance, etc.).
3. The automated repository workflows will classify your issue based on keywords and automatically route it to the `🧊 Backlog` column of the main Project Board.

## Backlog Grooming Process

- Issues are added continuously.
- Product Management and Engineering Leads review the `🧊 Backlog` column weekly.
- Items are tagged for Priority, Epic, and Size.
- Groomed items are moved to `🎯 Sprint Ready` for team pickup.
